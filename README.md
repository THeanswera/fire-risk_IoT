# fire-risk_IoT

Программа для автоматизированного расчета индивидуального пожарного риска по Приказу МЧС России №1140 с возможностью сравнения двух вариантов:

- традиционный расчет;
- расчет с учетом характеристик модульной IoT-СОУЭ.

Проект реализован на `Python` и `Streamlit` и предназначен для инженерной оценки сценариев развития пожара, параметров эвакуации и влияния интеллектуальной системы оповещения на итоговую величину ИПР.

## Назначение

Программа рассчитывает индивидуальный пожарный риск для объекта защиты по сценариям пожара `i` и группам людей `j`, используя нормативные зависимости Методики №1140. Для варианта с IoT-СОУЭ дополнительно вычисляется интегральный показатель эффективности системы, после чего автоматически пересчитываются:

- `K_СОУЭ,i`;
- `K_п.з,i`;
- итоговое значение ИПР;
- относительное снижение риска по сравнению с традиционным вариантом.

## Что умеет программа

- рассчитывать ИПР по сценариям и группам людей в соответствии с Методикой №1140;
- отдельно показывать традиционный расчет и расчет с учетом IoT-СОУЭ;
- вычислять коэффициенты `K_реак`, `K_IoT`, `K_связь`, `K_над`, `K_IoT_итог`;
- модифицировать `K_СОУЭ,i` через коэффициент `α`;
- пересчитывать `K_п.з,i` и итоговый риск для каждого сценария;
- отображать таблицы, метрики и сравнительные диаграммы;
- выгружать результаты в `CSV`, открываемые в `Excel`.

## Входные данные

### Таблица сценариев

Для каждого сценария задаются:

- `Q_n,i` — частота реализации пожароопасной ситуации, `год^-1`;
- `t_пр,i` — время присутствия людей, `ч/сут`;
- `t_бл,i` — время блокирования путей эвакуации, `мин`;
- `K_ап,i` — коэффициент эффективности автоматических установок пожаротушения;
- признаки соответствия для `K_обн,i`, `K_СОУЭ,i`, `K_ПДЗ,i`.

По Методике №1140 коэффициенты `K_обн,i`, `K_СОУЭ,i` и `K_ПДЗ,i` принимаются строго как `0.8` или `0.0` в зависимости от выполнения условий.

### Таблица групп людей

Для каждой группы задаются:

- привязка к сценарию `i`;
- наименование группы;
- `t_p,i,j` — расчетное время начала эвакуации;
- `t_н.э,i,j` — время неорганизованного этапа;
- `t_ск,i,j` — время существования скоплений.

### Параметры IoT-СОУЭ

В боковой панели настраиваются:

- показатели адаптивности: сенсоры, маршрутизация, автоматизация, время реакции;
- показатели надежности модулей: обнаружение, каналы связи, логика, оповещение, питание;
- вероятность киберинцидента `p_cyber`;
- эмпирический коэффициент вклада `α`.

## Результаты расчета

На экране одновременно отображаются:

- `R` для традиционного варианта;
- `R` для варианта с IoT-СОУЭ;
- абсолютная разница;
- процент снижения риска;
- соответствие нормативному условию `R <= 1e-6 год^-1`;
- промежуточные коэффициенты по сценарию и по IoT-модели;
- таблицы детализации по группам и итоговое агрегирование.

Дополнительно программа показывает сравнительные формулы и диагностический разбор расчета `P_э,i,j`.

## Методическая основа

В основе проекта лежат:

- Приказ МЧС России №1140 от 14.11.2022;
- разработанная авторами методика учета характеристик IoT-СОУЭ при расчете ИПР;
- материалы реферата к программе для ЭВМ:
  `«Автоматизированный расчет индивидуального пожарного риска с учетом характеристик модульной СОУЭ на основе технологии интернета вещей (IoT-СОУЭ)»`.

## Технологии

- `Python 3.12`
- `Streamlit 1.37.1`
- `pandas 2.2.2`

## Установка

```powershell
py -3.12 -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

## Запуск

```powershell
streamlit run app.py
```

После запуска в браузере откроется интерфейс приложения с таблицами сценариев, групп и панелью настройки IoT-СОУЭ.

## Структура проекта

- `requirements.txt` — зависимости проекта;
